# Scene reference images

This document explains how reference images for "real" scenes (the
hand-written, exact-prompt scenes in `constants/scenes.ts`, as opposed to the
100 procedurally generated filler scenes) are stored and wired up.

## What this folder is for

`assets/images/scenes/` stores the example preview image for each real scene
— a photo generated by running that scene's exact prompt (from
`constants/scenes.ts` / `docs/PROMPTS.md`) through Gemini with the reference
model photo. It's what `SceneCard` / `MasonrySceneCard` show as the card
background once available, instead of the gradient + emoji placeholder.

## Naming

Each filename matches the scene's `id` in `constants/scenes.ts`, using the
`exampleImage` field's basename as the source of truth, e.g.:

| Scene id                  | File                                          |
|----------------------------|-----------------------------------------------|
| `lamborghini_pov`          | `assets/images/scenes/lamborghini-pov.jpg`     |
| `dubai_jetski`             | `assets/images/scenes/dubai-jetski.jpg`        |
| `dubai_shopping_lambo`     | `assets/images/scenes/dubai-shopping-lambo.jpg`|
| `girlfriend_jet_bugatti`   | `assets/images/scenes/girlfriend-jet-bugatti.jpg` |
| `highrise_pool`            | `assets/images/scenes/highrise-pool.jpg`       |

The reference model photo used to generate all of these lives at
`assets/images/scenes/reference-model.jpg`.

## Format

- JPG
- 9:16 aspect ratio
- Minimum 1080x1920px

## How images get added (manual only)

Images are added **manually by the project owner only**. Nothing in this
codebase auto-generates, fetches, or modifies these files.

React Native's bundler (Metro) requires static `require()` paths, so dropping
a file into this folder alone isn't enough to make it show up — you also need
to add one line to `constants/sceneImages.ts` mapping the scene id to that
file:

```ts
lamborghini_pov: require('../assets/images/scenes/lamborghini-pov.jpg'),
```

Once that entry exists, `SceneCard` and `MasonrySceneCard` automatically use
the real image instead of the gradient + emoji fallback — no other code
changes needed.

## Do not

- Do not auto-generate or batch-fetch images into this folder.
- Do not modify the prompts in `constants/scenes.ts` / `docs/PROMPTS.md` to
  match an image — the prompt is the source of truth, the image is the
  output.
- Do not add scenes here that aren't already defined in
  `constants/scenes.ts`.
